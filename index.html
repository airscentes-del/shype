Interface

Launch, swap, route perps and track the vault.

Swap

Jupiter-routed swaps

Prepared for Jupiter Swap integration. Until wallet routing is connected, this page does not create transactions.

Fee path

When swap routing is live, supported integrator fees can route to a SHYPE fee account and appear in the vault model.

  • No custody of user funds
  • User signs every transaction
  • Fees only where protocol integration supports it

Perps

External leverage routing

Perps are not run by SHYPE. This interface can route users to existing Solana perp venues. Supported markets and leverage depend on the venue.

Jupiter PerpsOpen external venueDriftOpen external venue

Risk

Leverage trading can liquidate collateral. SHYPE should show this clearly before routing any user into perps.

Stake

Stake-to-support

No APY is advertised. Future staking should be used for trust score, ranking, vault access and optional variable rewards from real activity.

Staking opens after SHYPE launch and contract review.

Planned mechanics

  • Lock launched tokens to increase trust score
  • Unlock badges for community-locked launches
  • Variable claims only from real protocol fees

Launch

Pump.fun launch flow

SHYPE can start as a launch index: creators launch externally, then submit the mint for a SHYPE profile, swap route and vault tracking.

Principle

Non-custodial

SHYPE should never hold user collateral for trades. Users connect a wallet and sign transactions directly.

Routing

External liquidity

Swaps and perps should use established Solana protocols instead of SHYPE running its own matching engine at launch.

Status

Pre-launch

Token data, vault addresses and fee counters stay empty until the SHYPE token and real integrations exist.
