Documents

SHYPE is being built as a Solana-native app interface for token launches, swap routing, perps access, and transparent fee-flow experiments.

Current status

The project is currently a static GitHub Pages interface. Wallet actions must be signed by the user. SHYPE does not custody funds, request seed phrases, or show fake balances.

App modules

  • Launch: prepare token metadata, links, image, and routing model before handoff to a launch venue.
  • Swap: route token quotes through Jupiter-compatible infrastructure inside a SHYPE-styled interface.
  • Perps: prepare venue, market, collateral, and leverage context for future supported venue adapters.
  • Vault: show the intended fee-routing model before real vault accounts are deployed.

Fee-routing draft

  • 40% SHYPE main-token route.
  • 25% launch liquidity support.
  • 25% event or community voting pool.
  • 10% operations treasury.
